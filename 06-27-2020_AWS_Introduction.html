Key Points
  • Regions
    • Geographical area in the world where AWS resources located
    • Consist 2 or more Availability Zone

  • Availability Zones
    • Data center

    • May be collection of data centers
  • Edge Locations
    • Content Delivery Network end-point for CloudFront
    • Cache locations for some types of files like video or image files

Network and Content Delivery
  • VPC
    • Virtual Private Cloud
    • Virtual Data Center
  • Route 53
    • AWS DNS service
    • Register domain using route 53
    • 53 is DNS port
  • Cloud Front
    • Part of content delivery network
    • Caching
    • Consist of edge locations
  • Direct Connect
    • Connecting office/physical data center to AWS data center using physical line/dedicated telephone line instead of internet

Compute
  • EC2
    • Elastic Compute Cloud
    • Virtual machines in cloud
  • EC2 Container Service
    • container management service
    • supports docker containers
  • Elastic Beanstalk
    • If you do not know anything about AWS but want to deploy the code
    • Directly upload code and AWS will look the code and provision underlying infrastructure required
  • Lambda
    • Server-less virtual machine
    • Upload the code and code responds to events
    • No underlying host


Key Points
  • How to build VPC
  • Direct Connect
  • No EC2 container service in exam

Topic: Storage, Databases, Migration & Analytics
Date: 06/27/2020


Storage
  • S3
    • Simple Storage Service
    • Virtual disks to store objects (any type of files)
    • Not used to install any software
  • Glacier
    • Archive files from S3
    • Used if no instance access of files are required
    • Used mainly for data archival
  • EFS
    • Elastic File Service
    • File based storage as compared to S3 which is object based storage
    • Place where we can install DB or applications
    • Share volume to multiple VM
  • Storage Gateway
    • Way of connecting S3 to on premise data center

Databases (IMP)
  • RDS
    • Relational Database Service
  • DynamoDB
    • NoSQL database
  • Redshift
    • Data Warehousing solution
    • Only query to run reports as queries cannot be run on production DB as it slows the production DB
  • Elasticache
    • Way of caching in cloud
    • To reduce load from database

Migration
  • Snowball
    • Import/Export data from/to physical disk to S3
    • To move TB of data to cloud
    • Standard Physical device (briefcase like) instead of different type of disks like SATA/PATA etc.
  • DMS
    • Database Migration Service
    • Migrate on-premise database to AWS cloud
    • Migrate existing DB from one region to other
    • No production downtime
    • Conversion of DB(Oracle to Aurora etc.) also possible using DMS
  • SMS (Server Migration Service)
    • Database migration service like service but for virtual machines
    • VMWare virtual machines

Analytics
  • Athena
    • Allows SQL queries on S3
    • Turning flat files into searchable database
  • EMR
    • Elastic MapReduce
    • Big data processing
    • Process large amount of data
    • Mainly Hadoop framework used
  • Cloud Search
    • Mainly used to implement search capability for the website
    • Search engine for application or website
    • Fully managed service by AWS
  • Elastic Search
    • Mainly used to implement search capability for the website
    • Search engine for application or website
    • Used open source framework
  • Kenesis
    • Use for analysing real-time data
    • Capture and store TB of data per hour
  • Data Pipeline
    • Used to move data from one place to another
    • E.g.: From S3 to DynamoDB
  • Quick Sight
    • Business analytics tools
    • Helps to create visualization and rich dashboards for data
    • Can be used to analyse data in S3, DynamoDB, RDS, Redshift etc.

Key Points
  • S3 (IMP)
  • Storage Gateway (Architect, not in Developer as much)
  • Dynamo DB (Developer Exam IMP)
  • Redshift (Theory mainly)
  • Elasticache (Architect exam, Developer Exam IMP)
  • Snowball (Architect Exam, not much in Developer Exam)
  • EMR (Architect Exam, not much in Developer Exam)
  • Kenesis (Architect Exam)

Topic: Security, Management Tools, Application Services, Developer Tools, Mobile Services & IoT
Date: 06/27/2020


Security & Identity
  • IAM
    • All kind of administration for identity and access management
  • Inspector
    • Agent that can be installed on VMs
    • Inspects the VMs and does security recordings of VMs
  • Certification Manager
    • Provides free SSL certificates for domain names
  • Directory Service
    • Way of using active directory with AWS (Connect active Microsoft active directory to AWS)
  • WAF
    • Web Application Firewall
    • Give application level protection to website
    • Firewall for network level protection however, WAF for application level protection like SQL injection, cross-site scripting etc. at application layer
  • Artifacts
    • Documentations in AWS console
    • Used to get Compliance documents

Management Tools (IMP)
  • Cloud Watch
    • Monitor performance of AWS environment
    • E.g.: Monitor EC2 for CPU/RAM/Disk utilization etc.
  • Cloud Formation
    • Turning infrastructure into code
    • E.g.: Instead of having physical firewalls/network switches/ load balancers/servers, we can cloud formation templates which can be provisioned using command line
  • Cloud Trail
    • Auditing AWS resources
    • E.g.: If user is created in IAM, it will be logged
    • Auditing changes to AWS environment
  • Opsworks
    • Automating deployment using chef
  • Config
    • Automatically monitors environment and sends notifications based on policies set
    • Different from Cloud Trail in way that Cloud Trail only logs/records changes, Config actively monitors and notifies about changes
  • Service Catalog
    • Designed for large enterprise
    • Build out specific set of service that enterprise wants to authorize and not authorize enterprise wide
  • Trusted Advisor
    • Tips for performance automation or such type of automations
    • Automated way of scanning the environment and give tips based on that

Application
  • Step Functions
    • Way of visualizing what's going on in the application
    • E.g.: What different micro services it is using
  • SWF
    • Simple Workflow Service
    • Co-ordinate automated and human led tasks
  • API Gateway
    • Tool that allows create, publish, maintain, monitor, secure API at scale
    • Doors to access back-end data
  • AppStream
    • Streaming desktop applications to users
  • Elastic Transcoder
    • Convert video to format which is suitable for all devices

Developer Tools
  • CodeCommit
    • Basically github
    • Store code into cloud
  • CodeBuild
    • Pay by minute
    • Compiling code into different environments
  • CodeDeploy
    • Deploying code to EC2 instance
    • Does in automated and regulated manner
  • CodePipeline
    • Keep track of different versions of code

Mobile
  • Mobile Hub
    • Add, configure, design features for mobile app
    • Console for mobile app development
  • Cognito
    • Signup and sign-in to the app using social identity providers
  • Device Farm
    • Enables to improve quality of mobile apps by quickly and securely testing on many mobile devices
    • Test your app in physical devices in AWS datacenters
  • Mobile Analytics
    • Lets you collect and analyse app usage data of mobile app
  • Pinpoint
    • Understand and engage users of application
    • Much like google analytics with targeted marketing campaign

Business Productivity
  • WorkDocs
    • Storing work related document in cloud
    • Store in S3 with security
  • WorkMail
    • provides mail server for sending or receiving mail

Internet Of Things
  • iOT
    • Way of keeping track of many devices

Desktop & App Streaming
  • WorkSpace
    • Kind of VDI
    • Desktop in the cloud
  • AppStream 2.0
    • Way of streaming desktop applications to users

Key Points
  • IAM (Most IMP)
  • Directory Service (mainly in Solution Architect Exam)
  • WAF (Not in any exam yet)
  • Cloud Formation (Most IMP for Solution Architect not Exam but practical)
  • Cloud Trail (IMP Security Exam)
  • Config (IMP Security Exam)
  • Trusted Advisor (Mainly in Solution Architect Associate Exam)
  • SWF (IMP)
